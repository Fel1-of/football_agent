# Лабораторная работа 1 — Определение местоположения игрока

Реализация на Python 3 без внешних зависимостей.

## Что делает агент

- Подключается к `rcssserver` и отправляет начальную позицию `move`.
- После `play_on` вращается с заданной скоростью `turn`.
- По сообщениям `see` вычисляет координаты игрока по флагам.
- Выводит в консоль координаты игрока, мяча и противника (если видимы).

## Запуск

```bash
python3 app.py --team teamA --x -15 --y 0 --rotation 20
python3 app.py --team teamB --x 15 --y 0 --rotation 0
```

Перед запуском агентов поднимите `rcssserver` (и при необходимости `rcssmonitor`).

## Замечания по методичке

1. **Координаты флагов по оси y.** В методических указаниях у флагов перепутан знак по оси y. В коде используются координаты, согласованные с визуализацией поля в мониторе (в настройках можно включить отображение флагов; при наведении мыши показываются координаты). Иначе при спавне, например, по y=5 на мониторе в терминале выводилось бы -5.

2. **Вратарь (гл. 3, дерево решений).** В методичке указано «на расстоянии до 2 мяч можно поймать (catch)» — число 2 неверно. Корректная формула:
   - **catchable_distance = player_size + ball_size + catchable_area**
   - По умолчанию: 0.3 + 0.085 + 1.2 = **1.585**. Если мяч дальше, вратарь не поймает.
   - **kickable_distance** ≈ 0.5. Если вратарь поймал мяч на расстоянии ~1.5, удар может быть недоступен — через такт мяч отпускается без удара.
