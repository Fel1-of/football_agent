# Лабораторная работа 3 — Деревья решений для игрока и вратаря

Реализация на **Python 3** без внешних зависимостей.

## Требования

- Запущенный сервер виртуального футбола `rcssserver`
- При необходимости монитор `rcssmonitor`
- Python 3.6+

## Запуск

```bash
# Два полевых игрока в одном звене
python3 app.py --team teamA --x -20 --y 0 --rotation 20 --squad-size 2
python3 app.py --team teamA --x -24 --y -6 --rotation 20 --squad-size 2

# Вратарь справа
python3 app.py --team teamA --x 48 --y 0 --rotation 20 --goalie
```

Или скриптом:

```bash
./start.sh
```

## Что реализовано

- В `controller.py` добавлен менеджер исполнения дерева решений `DecisionTreeManager`.
- Полевой игрок использует дерево решений для движения по маршруту и для координации в звене из 2 или 3 игроков.
- Роль в звене выбирается автономно по своему `unum` и видимым партнёрам.
- Лидер идёт по маршруту из флагов и завершает атаку ударом по `gr`.
- Ведомые держат дистанцию и угол относительно ведущего: слева сзади для звена из двух, слева и справа сзади для звена из трёх.
- Вратарь защищает правые ворота, возвращается в центр ворот, ловит близкий мяч через `catch` и выбивает его в сторону левых ворот.

## Структура

- `app.py` — точка входа, аргументы `--team`, `--x`, `--y`, `--rotation`, `--goalie`, `--squad-size`
- `agent.py` — сетевой агент, обработка `init`/`see`/`hear`, передача восприятия в контроллер
- `controller.py` — деревья решений для полевого игрока и вратаря
- `position.py` — геометрия и извлечение мяча, своих и чужих игроков из `see`
- `flags.py` — координаты флагов и извлечение видимых ориентиров
- `msg.py` — разбор сообщений сервера
- `socket_client.py` — UDP-клиент

## Примечание

После сообщения `goal_*` маршрут полевых игроков начинается заново.
