# Лабораторные работы 1-2 — Позиционирование и движение по маршруту

Реализация на **Python 3** (без внешних зависимостей).

## Сравнение с lab1_friend (что есть у друзей и мы переняли)

| Что есть у друзей | У нас после рефакторинга |
|-------------------|--------------------------|
| Константы поля `FIELD_WIDTH`, `FIELD_HEIGHT` в одном месте | **config.py**: `FIELD_WIDTH`, `FIELD_HEIGHT`, `EPS` |
| Класс `SocketClient` (send/receive/close, timeout) | **socket_client.py**: класс `SocketClient` |
| Ожидание ответа `init` от сервера (таймаут 5 сек) | **Agent.connect()** ждёт init |
| Цикл в агенте: `run(start_pos, rotation)` и `stop()` с `(bye)` | **Agent.run()** / **Agent.stop()** |
| Вывод координат **мяча** | **Agent._process_see()** выводит `[Мяч] x=... y=...` |
| Обработка hear: play_on, kick_off, goal_ | **Agent._process_hear()** |
| Флаг `--goalie` | **app.py** `--goalie` |
| Документация команд сервера | **docs/server_commands.md** |

У друзей также: парсер возвращает плоский список (не вложенные dict), отдельная функция `compute_object_position` через теорему косинусов (расстояние объект–флаг), скрипт **start.sh** для запуска двух агентов.

## Требования

- Запущенный сервер виртуального футбола (`rcssserver`) и при необходимости монитор (`rcssmonitor`)
- Python 3.6+

**Установка сервера на macOS:** см. [docs/INSTALL_MAC.md](docs/INSTALL_MAC.md).  
**Установка сервера на Ubuntu:** см. [docs/INSTALL_UBUNTU.md](docs/INSTALL_UBUNTU.md).

## Запуск

```bash
# Игрок teamA (наша команда)
python3 app.py --team teamA --x -15 --y 0

# Игрок teamB (противник), в другом терминале
python3 app.py --team teamB --x 15 --y 0
```

Или скриптом: `./start.sh` (предварительно запустите rcssserver и rcssmonitor).

В мониторе: **Kick off** → **Play on**. В консоли выводятся координаты игрока, мяча и противника (если видны).

## Структура

- `app.py` — точка входа, аргументы (--team, --x, --y, --rotation, --goalie), вызов agent.run/stop
- `agent.py` — класс **Agent**: connect (ожидание init), move/turn, run/stop, обработка see/hear, вывод координат игрока, мяча, противника
- `socket_client.py` — класс **SocketClient**: send/receive/close, таймаут
- `controller.py` — объект **Controller** с маршрутом действий (`flag`/`kick`) для ПР2
- `config.py` — константы **FIELD_WIDTH**, **FIELD_HEIGHT**, **EPS** (без «магических чисел» в коде)
- `msg.py` — разбор сообщений сервера (see, hear, init)
- `flags.py` — координаты флагов поля, извлечение видимых флагов из see
- `position.py` — функции геометрии: позиция по 2/3 флагам, угол тела, координаты объекта (мяч/игрок), **in_field()** использует константы из config
- `docs/server_commands.md` — краткая шпаргалка по протоколу

## Что реализовано для практической работы 2

Используется базовая последовательность действий из методички:

```python
[
    {"act": "flag", "fl": "fc"},
    {"act": "kick", "fl": "b", "goal": "gr"},
]
```

Алгоритм движения соответствует методичке:

- команды формируются только после `play_on`;
- если целевой флаг/мяч не виден, агент делает `turn 90`;
- при видимой цели агент поворачивается к ней и бежит (`dash`);
- флаг считается достигнутым при `dist < 3`, после этого переход к следующему действию;
- для `kick`: мяч целевой при `dist <= 0.5`, удар по воротам при видимых воротах, иначе `kick 10 45`;
- после сообщения `goal_*` маршрут начинается заново с первого шага.

## Задание (методичка, ПР1)

1. Начальные координаты и скорость вращения из командной строки, команда `move`.
2. После начала игры — вращение с заданной скоростью (turn на каждом такте).
3. По сообщениям `see` вычислять свои координаты по ориентирам (флагам) и выводить в консоль.
4. На поле разместить второго игрока (команда противника).
5. Одновременно вычислять и выводить координаты видимого игрока противника.
