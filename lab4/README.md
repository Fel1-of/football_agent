# Лабораторная работа 4. Координация агентов

## Задание

1. **Согласованная игра двух агентов (teamA):**
   - **Passer** — игрок, который выходит к центральному флагу своей половины (fplc), находит мяч, подбирается к нему и отдаёт пас напарнику (kick в сторону ближайшего тиммейта) и подаёт голосовую команду **"go"** (say).
   - **Scorer** — игрок, который занимает позицию у чужих ворот (движение к угловому флагу fplb, затем к флагу у ворот fgrb), по команде **"go"** от пасующего бежит к мячу и бьёт по воротам (kick в направлении флага ворот gr/gl).

2. **Поведение задаётся деревом решений:** у каждого агента своё дерево (узлы: выполнить действие, проверить условие, выдать команду серверу). Контроллер на каждом такте обновляет данные (видимые объекты, позиция, сообщения от рефери и тиммейтов) и запускает обход дерева, в итоге отправляется одна команда (turn, dash, kick и т.д.).

3. **Команда противника (teamB):** два статичных защитника у своих ворот (у штанг). Они занимают заданную позицию (move), после play_on держатся на месте (dash 0), чтобы имитировать защиту и предотвращение офсайда.

4. **Цель:** реализовать координацию пасующий → забивающий через обмен сообщением "go" и показать работу деревьев решений для выбора действий агентов.

---

Реализация: passer и scorer (teamA) через дерево решений, статичные защитники (teamB). В `src/` скопированы модули lab1 (socket_client, msg, flags, position, config) — без импорта из lab1, всё самодостаточно.

## Запуск

1. Запустите rcssserver (и при необходимости rcssmonitor).
2. Из каталога `lab4/`:
   ```bash
   chmod +x start.sh
   ./start.sh
   ```
3. В мониторе нажмите KICK OFF.

Отдельно:
- Passer: `python3 src/main.py --team teamA --role passer --x -15 --y 0`
- Scorer: `python3 src/main.py --team teamA --role scorer --x -15 --y 10`
- Защитник: `python3 src/main_defender.py --team teamB --x -48 --y 5`

## Структура

- **decision_tree.py** — дерево решений (exec / condition / command).
- **dt_manager.py** — менеджер данных для дерева (видимые объекты, флаги ворот, тиммейты).
- **controller.py** — выбор дерева по роли (passer / scorer / goalie), вызов decide().
- **passer_dt.py**, **scorer_dt.py** — деревья: движение к точке, к мячу, пас/гол, обмен сообщением "go".
- **agent.py** — подключение, сбор visible_objects из see, расчёт позиции по флагам (position, flags из src/), вызов controller.decide(), say/kick/turn/dash.
- **main_defender.py** — статичный защитник: move к позиции, при play_on — dash 0.
