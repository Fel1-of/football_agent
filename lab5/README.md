# Лабораторная работа 5. Решение целевой задачи в условиях противодействия

Реализация на Python 3: интерпретатор **временного автомата** + два автомата:
- атакующий игрок (`attacker`);
- вратарь-защитник (`goalie`).

Сценарий: `teamA` пытается забить, `teamB` защищает свои ворота и перехватывает/выносит мяч.
Стиль проекта приведен к первым лабораторным: рабочие файлы лежат в корне `lab5/`, точка входа — `app.py`.

## Что реализовано по заданию 5.4

1. Программа обработки временного автомата:
   - файл `timed_automaton.py`;
   - состояния (locations), переходы (transitions), guards, сброс часов (reset), действия (action).
2. Временной автомат атакующего:
   - файл `attacker_ta.py`;
   - поиск мяча, подбегание, выравнивание на ворота, удар, восстановление.
3. Временной автомат вратаря:
   - файл `goalie_ta.py`;
   - удержание позиции у своих ворот, трекинг мяча, `catch`, `kick` (вынос), возврат в стойку.
4. Контроллер, переключающий автоматы по роли:
   - файл `controller.py`.

## Запуск

1. Запустить `rcssserver` и `rcssmonitor`.
2. Из каталога `lab5/`:
   ```bash
   chmod +x start.sh
   ./start.sh
   ```
3. В `rcssmonitor` нажать `Kick off`, затем `Play on`.

Ручной запуск:

- Атакующий:
  ```bash
  python3 app.py --team teamA --role attacker --x -20 --y 0
  ```
- Вратарь:
  ```bash
  python3 app.py --team teamB --role goalie --x 50 --y 0
  ```

## Структура

- `timed_automaton.py` — интерпретатор временного автомата.
- `ta_manager.py` — доступ к видимым объектам (дистанция/угол/видимость), флагам ворот.
- `attacker_ta.py` — автомат атакующего.
- `goalie_ta.py` — автомат вратаря.
- `controller.py` — выбор автомата и один шаг управления на такт.
- `agent.py` — подключение к серверу, парсинг `see/hear`, передача данных в контроллер.
- `app.py` — входная точка, выбор роли `attacker|goalie`.

## Как проверять требование вероятностей

По заданию требуется показать:
- вероятность перехвата > вероятность гола;
- обе вероятности не равны нулю.

Практически:
1. Проведите серию запусков (например, 20–30 розыгрышей).
2. Посчитайте:
   - `N_goal`: число сообщений `goal_...`;
   - `N_intercept`: число эпизодов, где вратарь поймал/выбил мяч до гола.
3. Оцените:
   - `P(goal) = N_goal / N_total`;
   - `P(intercept) = N_intercept / N_total`.

В текущей настройке автомата вратарь намеренно агрессивнее в зоне мяча, поэтому обычно выполняется `P(intercept) > P(goal)`, при этом голы остаются возможными.
